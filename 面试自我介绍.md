# 面试自我介绍 — 金融研报 Agent 项目

## 1 分钟版（自我介绍时用）

> 我做了一个金融研报 Agent，覆盖 A 股 5000+ 和公募基金 10000+。
>
> 它不只是问答——核心是**自主运行**。每个工作日早上 10 点和下午 2 点半，Agent 自动拉取大盘指数、热点板块和我持仓的 8 支基金数据，调用 DeepSeek 模型生成分析报告，包括行业集中度诊断、持仓重叠检测和调仓建议，然后通过 Server 酱推送到我微信。
>
> 整个过程不需要我打开电脑、不需要我点按钮。这是 LLM 做不到的事情。
>
> 技术栈是 Python + Streamlit + Anthropic SDK，Agent Loop 是我手写的，没用 LangChain 这种重框架。还支持多用户——每个人配置自己的持仓和推送 key，互不干扰。

## 面试官可能追问 & 你的回答

### Q: Agent 和普通 LLM 的区别？
> 普通 LLM 只能一问一答。Agent 有工具调用能力——我的系统里有 7 个工具：搜股票、查行情、拉 K 线、搜基金、查估值、净值历史、基金详情。Agent 自己决定什么时候调哪个工具、调几次，不需要我写 if-else。

### Q: 为什么不用 LangChain？
> 我手写了 Agent Loop 核心逻辑，就是一个 while 循环：发 prompt → 模型返回 tool_use → 我执行工具 → 结果塞回对话 → 循环直到模型输出最终文本。一共 50 行代码。理解底层之后再用框架很容易，但反过来不行。

### Q: 定时报告怎么实现的？
> GitHub Actions 定时触发 Python 脚本 → 拉数据 → 调 Agent 生成报告 → Server 酱推送到微信。还加了交易日检测：周末跳过、节假日通过上证指数成交量判断（成交量为 0 即休市）。

### Q: 遇到什么挑战？
> 有两个。一个是 DeepSeek 的 Anthropic 兼容接口在 GitHub Actions 美国服务器上返回 404，排查后发现是终端模型返回了 ThinkingBlock 导致 SDK 解析失败，加了一层 block type 判断解决了。另一个是推送服务的选择——PushPlus 要实名认证、WxPusher 收不到消息，最后换了 Server 酱才稳定。

### Q: 如果重新设计会怎么做？
> 加一个真正的数据库，目前多用户靠 GitHub 上的 JSON 文件，用户要手动保存。如果有数据库，用户注册和持仓管理可以完全自动化。另外会把分析引擎和推送引擎拆成微服务。
